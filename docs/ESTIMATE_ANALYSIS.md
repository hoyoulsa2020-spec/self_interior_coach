# 견적 참고 분석 (Estimate Analysis) — 준비 문서

> 목표: [견적코치](https://xn--l89a913cepe12a.com/) 유사 **참고·1차 필터** (법적·계약적 “적정가 판정”이 아님)  
> 전제: 업체마다 양식이 다르므로 **오차·검수**를 전제로 한다.

## 1. 포지셔닝

- 화면/리포트에 **“AI 추정·내부 참고 구간·계약 전 전문가 확인 권장”** 문구 고정.
- “시장 평균” 표현은 **실제 데이터 근거가 있을 때만** (없으면 “참고 구간”).

## 2. 단계별 로드맵

| 단계 | 내용 |
|------|------|
| **MVP-1** | 엑셀(xlsx) 업로드 → 셀 텍스트 추출 → 관리자에서만 조회·수동 매핑 연습 |
| **MVP-2** | `category` 연동 + 동의어 테이블로 대공정명 정규화 → 금액 합계 검증 플래그 |
| **MVP-3** | `estimate_reference_bands` 기준과 비교 → 낮음/구간/높음 + 신뢰도 |
| **MVP-4** | (선택) PDF 텍스트 추출 → 동일 파이프라인 |
| **MVP-5** | (선택) 스캔 PDF/이미지 OCR → 비용·오류율 감수 |

## 3. 파이프라인 (요지)

1. 파일 저장 (Supabase Storage 또는 임시 서버 디스크)
2. 엑셀 → 2차원 배열 / “행 단위 문자열”
3. 규칙 + (필요 시) LLM: 행 → 대공정 후보 + 금액
4. 동의어·키워드로 `category` 매핑
5. (선택) 면적(㎡)으로 ㎡당 환산 후 기준 밴드와 비교
6. 결과 JSON 저장 + `needs_review` 플래그
